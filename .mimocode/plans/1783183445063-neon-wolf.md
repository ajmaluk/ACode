# Deep Scan Fix Plan — All Remaining Issues

## Priority 1: Critical Bugs

### 1a. Keyboard shortcut conflicts between keydown handler and native menu (macOS)
**Files:** `App.tsx`, `lib/nativeMenu.ts`

Three shortcuts fire BOTH the keydown handler AND the native menu action, causing double-fire:

| Shortcut | keydown action | native menu action | Result |
|----------|---------------|-------------------|--------|
| Cmd+N | `newChat()` | `file.new-file` | Both fire — new chat AND new file |
| Cmd+Shift+P | `togglePalette()` | `help.command-palette` | Palette toggles twice — appears closed |
| Cmd+E | toggle view mode | `view.agent-mode` | Double-set — unpredictable mode |

**Fix:** In `App.tsx` keydown handler, skip handling these shortcuts when on macOS (the native menu handles them). Add a `platform()` check:
```typescript
if (mod && e.key.toLowerCase() === "n" && !isTyping(e.target)) {
  if (platform() === "mac") return; // native menu handles this
  e.preventDefault();
  useChat.getState().newChat();
  if (useUI.getState().viewMode !== "chat") useUI.getState().setViewMode("chat");
  return;
}
```

### 1b. Platform detection inconsistency — 5 locations use raw `navigator.userAgent`
**Files:** `dalamAPI.ts:793,2179,3241`, `TerminalPanel.tsx:307,314`

These use `navigator.userAgent.includes("Windows")` instead of the centralized `platform()` utility from `lib/platform.ts`. In Tauri v2, the user agent may not contain "Windows" in the expected format.

**Fix:** Replace all 5 with `import { isWindows } from "@/lib/platform"` and use `isWindows`.

### 1c. `revealInFinder` fallback uses macOS `open` on Linux
**File:** `dalamAPI.ts:2180`

The fallback is `const cmd = isWindows ? "explorer" : "open"`. The `open` command is macOS-only. Linux needs `xdg-open`.

**Fix:** `const cmd = isWindows ? "explorer" : isMac ? "open" : "xdg-open"`

### 1d. `run_command` and `run_preview` hardcode `bash` instead of using user's shell
**Files:** `dalamAPI.ts:3242`, `dalamAPI.ts:3654`

`run_command` uses `isWindows ? "powershell" : "bash"` but the user may have configured zsh or fish as their default shell. `run_preview` always uses `"bash"`.

**Fix:** Use `platform().defaultShell()` or read from the terminal store's active shell.

### 1e. `goBackChat` desynchronizes `messages` from `sessionMessages`
**File:** `useAppStore.ts:3779-3817`

`goBackChat` replaces `messages` with a historical snapshot but does NOT update `sessionMessages`. If `sendMessage` is called while in this state, the API receives the historical snapshot while the backend retains the full context, corrupting the conversation.

**Fix:** Either update `sessionMessages` to match the snapshot, or clear `activeSessionId` when navigating history (preventing sends while in history mode).

## Priority 2: High Bugs

### 2a. FTS5 search escape misses `\`, `:`, `|`
**File:** `memoryStore.ts:202`

The `escapeFts` function doesn't escape backslash, colon, or pipe — all FTS5 special characters.

**Fix:** Add these characters to the replacement regex.

### 2b. YAML escape misses `|`, `!`, `&`, `*`
**File:** `memoryStore.ts:347-356`

The `yamlEscape` function doesn't escape pipe (block scalar), exclamation (tags), ampersand (anchors), or asterisk (aliases).

**Fix:** Add `.replace(/\|/g, "\\|").replace(/!/g, "\\!").replace(/&/g, "\\&").replace(/\*/g, "\\*")`

### 2c. `parseRow` crashes on malformed tags JSON
**File:** `memoryStore.ts:804`

`JSON.parse(row.tags)` throws on malformed JSON, crashing the entire memory search.

**Fix:** Wrap in try/catch, fall back to `[]`.

### 2d. Missing abort check at top of `sendPrompt` while-loop
**File:** `dalamAPI.ts:1503-1514`

No `ac.signal.aborted` check at the top of the loop body. Aborted during tool-result processing wastes a full LLM API call.

**Fix:** Add `if (ac.signal.aborted) break;` after line 1505.

### 2e. Direct `useChat.setState` + raw localStorage in ChatView bypasses debounced persistence
**File:** `ChatView.tsx:440-452,493-508`

`handleResetConfirm` and `handleRestoreMessages` call `useChat.setState(...)` directly and write to `localStorage` with raw `setItem`, bypassing the store's debounced `savePersistedMessages()`. The pending debounced save may overwrite with stale data.

**Fix:** Use the store's `savePersistedMessages()` and `savePersistedSessionSummaries()` functions instead of raw localStorage.

## Priority 3: Medium Improvements

### 3a. Add `worker-src` and `frame-src` to CSP
**File:** `tauri.conf.json:26`

Missing `worker-src 'self' blob:` for Monaco workers and `frame-src` for the browser panel.

### 3b. Terminal theme doesn't adapt to light mode
**File:** `TerminalPanel.tsx:12-39`

`DALAM_TERM_THEME` is hardcoded dark. No light variant exists.

**Fix:** Create a light theme variant and switch based on app theme.

### 3c. `beforeunload` async flush never completes
**Files:** `trajectoryRecorder.ts:492-500`, `useAppStore.ts:816-817`

Both call async flush functions in `beforeunload` with fire-and-forget, losing up to 5s of data.

**Fix:** Use `navigator.sendBeacon` for critical data, or accept the data loss and document it.

### 3d. `BASH_ARITY` missing common multi-word commands
**File:** `agents.ts:141-187`

Missing: `docker compose`, `docker exec`, `docker build`, `gh pr`, `gh issue`, `kubectl apply`, `npm test`, `pnpm test`, `bun test`, `cargo test`.

### 3e. Tab close operations create N separate state updates
**File:** `EditorPane.tsx:543-580`

`closeOthers`, `closeAll`, etc. call `closeTab()` individually in a loop. With 20+ tabs, this creates 20+ re-renders.

**Fix:** Batch into a single `useWorkspace.setState({ openTabs: filtered })`.
