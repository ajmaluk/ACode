# Deep Codebase Scan & Fix Plan — Dalam AI IDE

## Scope
Full codebase review of the Dalam monorepo: TypeScript renderer (~100+ files), Rust backend (5 files), shared-types package, config files, and tests. Goal: find every real issue and fix them.

## Phase 1: Parallel Deep Scans (5 subagents)

### 1A. Core Lib Scan (lib/*.ts — non-test)
Scan all files in `apps/desktop/src/renderer/lib/` (excluding `__tests__/` and `.test.ts`):
- **agentRuntimeContract.ts** — contract correctness, type safety
- **agents.ts** — agent logic, error handling
- **cancellationToken.ts** — cancellation race conditions
- **changeStack.ts** — undo/redo logic correctness
- **chatUtils.ts** — chat utility bugs
- **codeIndex.ts** — indexing logic, memory leaks
- **connectors.ts** — connector patterns, error handling
- **constants.ts** — constant correctness
- **contextManager.ts** — context management bugs
- **costTracker.ts** — cost calculation accuracy
- **dalamAPI.ts** — API call safety, error handling
- **database.ts** — SQL injection, connection handling
- **diff.ts** — diff algorithm correctness
- **doneCriteria.ts** — completion detection logic
- **dreamAgent.ts** — agent loop correctness
- **dreamProposalPipeline.ts** — pipeline logic
- **editorHelpers.ts** — editor utility bugs
- **errorPatterns.ts** — error pattern matching
- **genes.ts** — gene system logic
- **highlight.ts** — syntax highlighting
- **hookBus.ts** — event bus correctness
- **hookListeners.ts** — listener lifecycle, memory leaks
- **instructions.ts** — instruction generation
- **mcpCache.ts** — cache invalidation
- **memoryGraph.ts** — graph data structure correctness
- **memoryStore.ts** — persistence, data integrity
- **memoryTypes.ts** — type definitions
- **metrics.ts** — metrics collection
- **modelRouter.ts** — routing logic
- **nativeMenu.ts** — native menu integration
- **pathUtils.ts** — path handling, security
- **platform.ts** — platform detection
- **regexCache.ts** — regex correctness, ReDoS
- **safetyTimer.ts** — timer correctness
- **security.ts** — security vulnerabilities (XSS, injection)
- **skillCrystallizer.ts** — skill crystallization logic
- **skills.ts** — skill system
- **storage.ts** — storage layer
- **stripXml.ts** — XML stripping correctness
- **tokenizer.ts** — token counting accuracy
- **toolExecutor.ts** — tool execution safety
- **toolSchemas.ts** — schema validation
- **trajectoryRecorder.ts** — recording correctness
- **types.ts** — type definitions
- **updater.ts** — update mechanism
- **verificationEngine.ts** — verification logic

**Focus**: Logic bugs, type errors, race conditions, memory leaks, security issues, dead code, incorrect error handling.

- **useAppStore.ts** — main store correctness

### 1C. Test Scan (all *.test.ts and __tests__/)
Scan all test files for:
- Incorrect assertions
- Missing edge cases
- Flaky test patterns
- Wrong test expectations
- Dead/orphaned tests

### 1D. Rust Backend Scan (src-tauri/src/*.rs)
Scan all Rust files:
- **main.rs** — initialization, error handling
- **lib.rs** — library exports, module structure
- **git.rs** — git operations, command injection
- **system.rs** — system calls, security
- **build.rs** — build script correctness

**Focus**: Unsafe code, unwrap() on Results, command injection, panic paths, memory safety.

### 1E. Config & Infrastructure Scan
Scan configuration files:
- **package.json** (root + apps/desktop) — dependency issues, script correctness
- **tsconfig.json** — compiler option issues
- **turbo.json** — pipeline configuration
- **pnpm-workspace.yaml** — workspace config
- **.prettierrc** — formatting config
- **tauri.conf.json** — Tauri configuration
- **Cargo.toml** — Rust dependencies

## Phase 2: Consolidate Findings
- Deduplicate issues across scanners
- Classify by severity: Critical (security/crash) > High (logic bugs) > Medium (code quality) > Low (style)
- Prioritize fixes

## Phase 3: Fix All Issues
- Fix critical and high severity issues first
- Fix medium severity issues
- Fix low severity issues
- Verify fixes don't break existing tests

## Expected Output
- Complete list of all issues found
- All issues fixed in-place
- Summary of what was found and fixed
