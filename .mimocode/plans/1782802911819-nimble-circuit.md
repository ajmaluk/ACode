# Plan: Major UI Overhaul — VS Code-style Layout + macOS Integration

## Summary
Transform Dalam from a simple 3-panel layout to a VS Code-style IDE with: activity bar, bottom terminal, macOS-native title bar integration, glass effects, and separated agent/editor modes.

---

## Phase 1: macOS Title Bar — Remove Duplicate, Use Native

**Problem:** On macOS, `tauri.conf.json` has `"decorations": true` (native title bar), but the TitleBar component still renders a custom bar below it with workspace name, mode switcher, controls — creating a double title bar.

**Fix:**
- **File:** `apps/desktop/src/renderer/components/editor/TitleBar.tsx`
  - On macOS (`isMac`), hide the entire custom TitleBar component
  - Move macOS-specific controls (mode switcher, theme, settings) into the native menu bar via `nativeMenu.ts`
  - The workspace name and "Open with" dropdown move to the sidebar header

- **File:** `apps/desktop/src/renderer/App.tsx`
  - Conditionally render `<TitleBar />` only on Windows/Linux
  - On macOS, the app content starts directly below the native title bar

- **File:** `apps/desktop/src/renderer/lib/nativeMenu.ts`
  - Add macOS menu items for: mode switcher (Agent/Editor), theme toggle, terminal toggle

---

## Phase 2: Activity Bar — Left Side VS Code-style Icons

**Problem:** Currently the sidebar is a flat list of workspaces/sessions. VS Code has an activity bar with icons on the far left.

**New component:** `apps/desktop/src/renderer/components/sidebar/ActivityBar.tsx`

Activity bar icons (top to bottom):
1. **Explorer** (files) — `FolderTree` icon — shows file tree
2. **Search** — `Search` icon — shows search panel
3. **Source Control** — `GitBranch` icon — shows git status (badge with change count)
4. **Agent** — `Sparkles` icon — shows chat sessions (the current sidebar content)
5. **Extensions** — `Puzzle` icon — placeholder for extensions

Bottom of activity bar:
- **Settings** — `Settings` icon

The activity bar is a 48px wide vertical strip. Clicking an icon switches the sidebar content panel.

**Modify:** `apps/desktop/src/renderer/store/useAppStore.ts`
- Add `activityBarTab: "explorer" | "search" | "scm" | "agent" | "extensions"` to UI state
- Add `setActivityBarTab` action

**Modify:** `apps/desktop/src/renderer/components/sidebar/Sidebar.tsx`
- Split into two modes based on `activityBarTab`:
  - `"explorer"` → File tree view (existing FileTree component)
  - `"agent"` → Current workspace/session list
  - `"search"` → Search input + results
  - `"scm"` → Git status view (extracted from RightPanel)
  - `"extensions"` → Placeholder

---

## Phase 3: Terminal — Move to Bottom Panel

**Problem:** Terminal is currently in the RightPanel. VS Code puts it at the bottom.

**New component:** `apps/desktop/src/renderer/components/terminal/BottomPanel.tsx`

- A resizable bottom panel (like VS Code's bottom panel)
- Contains tabs: Terminal, Output, Problems
- Default height: ~200px, resizable from 100px to 50%
- Toggle with `` Ctrl+` `` (backtick) like VS Code

**Modify:** `apps/desktop/src/renderer/App.tsx`
- Add a bottom `PanelGroup` (vertical) below the main horizontal layout
- The bottom panel is collapsible and contains `BottomPanel`

**Modify:** `apps/desktop/src/renderer/store/useAppStore.ts`
- Add `bottomPanelOpen: boolean`, `bottomPanelTab: "terminal" | "output" | "problems"`
- Add toggle/set actions
- Change `Ctrl+\` to toggle bottom panel instead of right panel

**Modify:** `apps/desktop/src/renderer/components/rightpanel/RightPanel.tsx`
- Remove the "Terminal" tab from the right panel
- Right panel now only has: Git, Diff, Review, Progress, Browser

## Phase 4: Glass/Transparent Effects

**Modify:** `apps/desktop/src/renderer/index.css`
- Add glass utility class:
```css
.glass-panel {
  background: rgb(var(--dalam-bg-secondary) / 0.85);
  backdrop-filter: blur(12px) saturate(150%);
  -webkit-backdrop-filter: blur(12px) saturate(150%);
}
```
- Apply to: sidebar, right panel, bottom panel, command palette
- Add subtle border glow on hover for interactive elements

**Modify:** `apps/desktop/tailwind.config.js`
- Add `backdrop-blur` custom values

## Phase 5: Theme Cleanup

**Problem:** Some components use hardcoded colors instead of CSS variables.

**Scan and fix all components** that use:
- Inline `style={{ color: "..." }}` with hardcoded hex
- Tailwind palette colors (`text-blue-400`, `bg-amber-400/10`) that don't respect theme
- Replace with `text-dalam-accent-primary`, `bg-dalam-accent-subtle`, etc.

**Key files to audit:**
- `EditorPane.tsx` — many hardcoded colors in code blocks
- `ActivityBlocks.tsx` — some hardcoded colors
- `SettingsModal.tsx` — some hardcoded colors
- `Sidebar.tsx` — some hardcoded colors

---

## Phase 6: Agent/Editor Mode Separation

**Current state:** Both modes share the same layout. The mode switcher in the title bar toggles `viewMode`.

**Improvements:**
- **Agent mode:** Full-width chat area, no file tabs, sidebar shows sessions
- **Editor mode:** File tabs visible, sidebar shows file tree, bottom panel available
- Both modes share the same workspace/project
- The mode switcher should be more prominent and visually distinct

**Modify:** `apps/desktop/src/renderer/App.tsx`
- In editor mode: show file tab bar below the title bar area
- In agent mode: hide file tabs, show a clean chat interface

**Modify:** `apps/desktop/src/renderer/components/editor/TitleBar.tsx`
- Improve the mode switcher styling — use a pill-shaped toggle with smooth animation
- On macOS: move mode switcher to the native menu
