## Phase 1: Critical Security Fixes (Do First)

### Fix 1: `open_url` Protocol Restriction
**File:** `toolSchemas.ts:92-94`
**Problem:** `z.string().url()` accepts `javascript:`, `file:`, `data:`, `blob:` protocols — XSS and local file disclosure vectors.
**Fix:** Add `.refine()` to whitelist only `http:`, `https:`, `mailto:`.
```ts
export const OpenUrlArgsSchema = z.object({
  url: z.string().url("must be a valid URL").refine(
    (url) => {
      try { return ["http:", "https:", "mailto:"].includes(new URL(url).protocol); }
      catch { return false; }
    },
    "Only http, https, and mailto URLs are allowed"
  ),
});
```

### Fix 2: MCP Tool Security Scan
**File:** `toolSchemas.ts:373-388`
**Problem:** MCP tools (`mcp_*`) skip `DANGEROUS_PATH_PATTERNS` and `DANGEROUS_COMMANDS` checks entirely.
**Fix:** Add generic security scan for MCP tools — iterate all string arg values against path patterns. Also check `get_disk_space` in the existing traversal block.
```ts
if (toolName.startsWith("mcp_")) {
  // ... existing plain-object checks ...
  // Security scan for string arg values
  for (const val of Object.values(args)) {
    if (typeof val === "string") {
      for (const pattern of DANGEROUS_PATH_PATTERNS) {
        if (pattern.test(val)) {
          return { valid: false, error: "Path not allowed" };
        }
      }
      const normalizedCmd = normalizeCommand(val);
      for (const dangerous of DANGEROUS_COMMANDS) {
        if (normalizedCmd.includes(normalizeCommand(dangerous))) {
          return { valid: false, error: `Dangerous command blocked: contains '${dangerous}'` };
        }
      }
    }
  }
  return { valid: true, args };
}
```

### Fix 3: Verification Engine Newline Escaping
**File:** `verificationEngine.ts:57`
**Problem:** Newlines in commands split into multiple shell commands. `echo hello\nrm -rf /` would execute two commands.
**Fix:** Add `.replace(/\n/g, '\\n')` to the escape chain.

### Fix 4: `search_files` Path Traversal Check
**File:** `toolSchemas.ts:415`
**Problem:** `search_files` uses `args.pattern` for the path check, but `pattern` is a search pattern (e.g., `*.ts`), not a file path. Also `get_disk_space` takes a `path` arg but isn't in the traversal-checked tools list.
**Fix:** Change traversal check to read `args.path` for `get_disk_space`, and skip `search_files` (it searches by pattern, not by user-provided path). Add `get_disk_space` and `launch_app` (for `cwd`) to the checked list.
```ts
const pathTools = ["read_file", "write_file", "edit_file", "list_dir", "grep_file", "get_disk_space", "reveal_in_finder"];
if (pathTools.includes(toolName)) {
  const path = String(args.path ?? "");
  // ... dangerous path check
}
// Also check launch_app's cwd
if (toolName === "launch_app") {
  const cwd = String(args.cwd ?? "");
  if (cwd) {
    for (const pattern of DANGEROUS_PATH_PATTERNS) {
      if (pattern.test(cwd)) return { valid: false, error: `Path not allowed: ${cwd}` };
    }
  }
}
```

## Phase 2: High-Priority Reliability Fixes

### Fix 5: Dangerous Command False Positive Prevention
**File:** `toolSchemas.ts:427-437`
**Problem:** `rm -rf /tmp/build` contains substring `rm -rf /` and gets blocked. Legitimate `/tmp` operations fail.
**Fix:** Use prefix + space/end matching instead of substring:
```ts
for (const dangerous of DANGEROUS_COMMANDS) {
  const normalizedDangerous = normalizeCommand(dangerous);
  if (cmd === normalizedDangerous || cmd.startsWith(normalizedDangerous + " ")) {
    return { valid: false, error: `Dangerous command blocked: contains '${dangerous}'` };
  }
}
```

### Fix 6: `isPrivateHost()` SSRF Hardening
**File:** `security.ts:6-33`
**Problem:** Missing DNS rebinding (`*.nip.io`), Unicode normalization, octal IPs (`0177.0.0.1`), IPv6-mapped IPv4 (`[::ffff:127.0.0.1]`).
**Fix:** Add checks for octal IPs, IPv6-mapped IPv4 prefix, and normalize hostname via `new URL()` before checking. Add async DNS resolution option.
```ts
// Add after existing checks:
if (/^0[0-7]+(\.[0-7]+)*$/.test(hostname)) return true; // octal IP
if (/^\[?::ffff:\d+\.\d+\.\d+\.\d+\]?$/i.test(hostname)) return true; // IPv6-mapped
// NIP.io / sslip.io DNS rebinding
if (/\.(nip\.io|sslip\.io)$/i.test(hostname)) return true;
```
