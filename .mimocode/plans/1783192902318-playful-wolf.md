# Dalam Desktop App — Deep Issue Fix Plan

## Summary

Deep scan of the entire codebase found **57+ issues** across all layers. This plan covers the highest-impact fixes organized into priority tiers.

## P0 — CRITICAL (Must Fix Immediately)

### C1. Diff Engine Produces Corrupted Output
- **Files:** `src/renderer/lib/diff.ts:170, 269-271`
- **Issues:** (D1) `myersDiffSimple` line-by-line fallback produces incorrect diffs for segments >200 lines. (D2) Chunk boundary misalignment: `oldPos += CHUNK_SIZE` and `newPos += CHUNK_SIZE` advance both by same amount regardless of actual line consumption, corrupting diff output for offset content.
- **Fix:** (D2) Track actual consumed lines per chunk instead of fixed CHUNK_SIZE. (D1) Replace the line-by-line fallback with a proper O(mn) Myers for segments 200-2000 lines.
- **Files to modify:** `diff.ts:224-271`

### C2. Dangerous Command Bypass via Whitespace
- **File:** `src/renderer/lib/toolSchemas.ts:385-393`
- **Issue:** (T4) `DANGEROUS_COMMANDS` check uses substring `includes()` which can be bypassed with extra whitespace (`rm  -rf  /`), prefixes (`rmlink -rf /`), or missing globs (`rm -rf /*`).
- **Fix:** Normalize whitespace (collapse multiple spaces, trim) before checking. Add `rm -rf /*` to the block list. Use word-boundary regex matching instead of `includes()`.
- **Files to modify:** `toolSchemas.ts:380-400`

### C3. closeToRight/closeToLeft Iterate While Mutating State
- **File:** `src/renderer/components/editor/EditorPane.tsx:555-573`
- **Issue:** (EP3) These functions iterate `openTabs` while calling `closeTab()` which mutates Zustand state. The iteration can skip tabs or produce incorrect results.
- **Fix:** Snapshot the tab list before iterating: `const tabsToClose = openTabs.slice(idx + 1).map(t => t.path);` then close each path in a loop.
- **Files to modify:** `EditorPane.tsx:555-573`

### C4. Rate Limit Backoff Reset Per Request Defeats Protection
- **File:** `src/renderer/lib/dalamAPI.ts:1500-1501`
- **Issue:** `sessionRateLimitErrors.set(sessionId, 0)` resets backoff at start of each `sendPrompt`. Rate limit protection is defeated across turns.
- **Fix:** Remove the reset. Let backoff persist until the session ends. Only reset on explicit user action or after a long quiet period (>5 min).
- **Files to modify:** `dalamAPI.ts:1500-1501`

### C5. Module-Level Maps Grow Unboundedly (Memory Leak)
- **File:** `src/renderer/lib/dalamAPI.ts:24-35`
- **Issue:** `activeControllers`, `sessionStartTimes`, `streamCallbacks`, `streamCleanups`, `fileWatchers`, `pendingDiffProposals`, `mcpHttpSessions`, `emittedSessionEnds`, `sessionRateLimitErrors` — all grow without bound. Cleanup is incomplete.
- **Fix:** Add periodic cleanup in the `finally` block (every 10 min). Clean `mcpHttpSessions` on session end. Cap `emittedSessionEnds` at 100 entries. Add TTL-based cleanup for `pendingDiffProposals`.
- **Files to modify:** `dalamAPI.ts:1918-1965`

## P1 — HIGH (Fix This Week)

### H1. Streaming State Leaks
- **File:** `src/renderer/store/useAppStore.ts:2177-2185`
- **Issue:** `_pendingChanges` array has no cap and grows unboundedly during streaming. `pendingActivities` capped at 500 but activities are heavyweight objects.
- **Fix:** Cap `_pendingChanges` at 200 entries. Reduce `pendingActivities` cap to 100. Clear `_pendingChanges` in `message-end`.
- **Files to modify:** `useAppStore.ts:2177-2185, 3040-3129`

### H2. Compaction Overwrites In-Flight Streaming State
- **File:** `src/renderer/store/useAppStore.ts:3520-3579`
- **Issue:** Tier 1 and Tier 2 compaction call `set()` to replace `sessionMessages` while `appendStream` may be concurrently modifying the same array.
- **Fix:** Add a `_compactingSessions` guard check at the top of `appendStream`. If compaction is in progress, queue the stream update for after compaction completes.
- **Files to modify:** `useAppStore.ts:3520-3579, 2140-2170`

### H3. Anti-Thrashing Can Permanently Disable Compaction
- **File:** `src/renderer/store/useAppStore.ts:3481-3488`
- **Issue:** After 2-3 ineffective compaction attempts, the anti-thrashing mechanism effectively disables compaction for the session with no upper bound on retries.
- **Fix:** Add a `compactionRetryCount` per session. After 3 failures, force a full LLM compaction regardless of savings percentage. Reset retry count on successful compaction.
- **Files to modify:** `useAppStore.ts:3481-3604`

### H4. Terminal Process Leak on Rapid Tab Close/Reopen
- **File:** `src/renderer/components/terminal/TerminalPanel.tsx:121-220`
- **Issue:** (T1) Async `startIO` has a race condition where `procIdsRef` can be re-inserted after cleanup deleted the entry, leaking a terminal process.
- **Fix:** Use a generation counter instead of boolean `isCleanedUp`. Increment on each mount, check after every await.
- **Files to modify:** `TerminalPanel.tsx:100-230`

### H5. Streaming Content Re-renders Full Markdown Every Token
- **File:** `src/renderer/components/chat/ChatRendering.tsx:132-136`
- **Issue:** (C3) During active streaming, `closeIncompleteMarkdown` + full `react-markdown` re-parse runs on every delta. Causes frame drops for long responses.
- **Fix:** Debounce markdown re-render to max 10fps during streaming. Use a simple text rendering for the last 500 chars of streaming content, only render full markdown on `message-end`.
- **Files to modify:** `ChatRendering.tsx:132-136`

### H6. Monaco Editor Options Recreated Every Render
- **File:** `src/renderer/components/editor/Editor.tsx:172-220`
- **Issue:** (E3) The `options` prop is an inline object literal — new reference every render. Monaco applies all options on every re-render.
- **Fix:** Wrap in `useMemo` with proper dependency array. Only recreate when actual values change.
- **Files to modify:** `Editor.tsx:172-220`

### H7. Session Restore Race with Workspace Loading
- **File:** `src/renderer/store/useAppStore.ts:1360-1361, 471-484`
- **Issue:** `loadFileTree` at line 483 has no workspace-path guard. A stale workspace path's tree can overwrite the new workspace's tree during rapid workspace switching.
- **Fix:** Add `_workspaceLoadPath` guard to `loadFileTree` call, same as `_doLoadWorkspaceConfigAndSessions`.
- **Files to modify:** `useAppStore.ts:471-484`

### H8. Non-Atomic localStorage Fallback Pruning Loop
- **File:** `src/renderer/store/useAppStore.ts:819-841`
- **Issue:** On `QuotaExceededError`, the code attempts fallback pruning and writes to localStorage, but doesn't update `_pendingMessagesRef`. The next save cycle re-writes the original full data, hitting quota again.
- **Fix:** After successful fallback write, update `_pendingMessagesRef` with the pruned data.
- **Files to modify:** `useAppStore.ts:819-841`

### H9. Stale Closure in Error Handler Writes to Wrong Session
- **File:** `src/renderer/store/useAppStore.ts:2067-2100`
- **Issue:** `sendSessionId` (line 2067) and `sessionId` (line 2079) are different variables. If session switches between try and catch, `errorMsg` gets written to the wrong session.
- **Fix:** Use `sendSessionId` consistently throughout the catch block.
- **Files to modify:** `useAppStore.ts:2067-2100`

### H10. Sub-Agent Abort Listener Leak
- **File:** `src/renderer/lib/dalamAPI.ts:4174-4175`
- **Issue:** `signal.addEventListener("abort", onParentAbort)` added every iteration (up to 10x). If `streamChat` throws synchronously, the listener leaks.
- **Fix:** Move listener registration inside the try block, ensure cleanup in finally.
- **Files to modify:** `dalamAPI.ts:4170-4200`

### H11. Stream Buffer Flush on Abort
- **File:** `src/renderer/lib/dalamAPI.ts:382-508, 511-638`
- **Issue:** (D17, D18) `_tcArgBuffers` and `_anthropicToolBuffers` are not flushed when the stream is aborted. Partial tool calls are silently dropped.
- **Fix:** Flush remaining buffers in the `finally` block before releasing the reader.
- **Files to modify:** `dalamAPI.ts:503, 633`

### H12. DangerouslySetInnerHTML XSS Risk
- **File:** `src/renderer/components/chat/ChatRendering.tsx:198, 249`
- **Issue:** (C2) `hljs.highlight()` output passed to `dangerouslySetInnerHTML`. If highlight.js has a bug, crafted HTML could execute.
- **Fix:** Wrap highlight.js output in a sanitizer pass. Use `DOMPurify.sanitize()` on highlighted output before setting innerHTML.
- **Files to modify:** `ChatRendering.tsx:198, 249`
